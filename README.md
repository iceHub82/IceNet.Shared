<h1>IceNet.Shared</h1>

<p dir="auto">Solution containing Maui hybrid and Blazor Server apps with user authorization and with a shared Razor component library containing Layouts, Pages and Components.</p>

<p dir="auto">The aim for this solution is to have as much code reused across both applications in form of Layouts, Pages and Components. Effort has been made to make the Maui Blazor app and the Blazor Server app as "thin" as possible by putting shared code in the Razor component library. As Maui Blazor runs on either Desktop or Mobile and Blazor server runs in a browser, there were issues on how to have a shared login page that authorized users coming from either Desktop/Mobile or from a Web Browser. For authorizing users in Maui(Desktop and Mobile) the `AuthenticationStateProvider` is used and for web users are authenticated with cookies. Furthermore to persist users authorization in Desktop/Mobile, user info has been added to the Preference storages on the devices.</p>
